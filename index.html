git pull 你跟你的小伙伴同时修改了dev分支，并且同时push，这时会发生冲突，需要先用此命令将最新的提交取到本地，在本地合并，解决冲突，再push。

    小结

  1. 查看远程库信息，使用git remote -v；

  2. 本地新建的分支如果不推送到远程，对其他人就是不可见的；

  3. 从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交；

  4. 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；

  5. 建立本地分支和远程分支的关联，使用git branch –set-upstream branch-name origin/branch-name；

  6. 从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

URl和URI

URI:Uniform Resource Identifier.
URL:Uniform Resource Locator.
URN:Uniform Resouce Name.

URL时URI的一种，URL是URI，但URI不一定是URL，还可能是其他，比如URN.
维金百科如是说：URN就像一个人的名字，URL就像一个人的地址;URN确定了东西的身份，URL提供了找到他的方式。
目前用URI的说法应该比较专业 一点，URL这个术语正在被抛弃
比如：file.hp.com 这是URI,但不是URL 系统可能对很对协议和端口都做出正确的反应。http://file.hp.com 和 ftp://file.hp.com可能是完全不同的内容。

结语

If you don’t mind being “that guy”,URI is probably the more accurate term to use.But if you are in the linguist/“use what’s understood” camp ,feel free to go with URL.

html和xml和xhtml

  1. 首先先搞清楚什么是html？什么是xml？ 什么是html？
    简单的说， HTML是超文本指标语言的因为缩写，XML是扩展标记语言的缩写，XHTML是可扩展超文本标记语言，是一种置标语言，表现方式与超文本标记语言（HTML）类似，不过语法上更加严格。

  2. 他们又有什么关系呢？（html跟xhtml暂且统称为html）

    2.1 首先他们都是标记语言，是一种特殊的文本标记，用途当然是用于传输数据和现实信息了.

    2.2xml与html在形式上很相似，比如都有类似这样的符号字符,在早期的网页设计中重要是用html来表示信息，浏览器可以解释执行它。html的最高版本是5，现在基本上被XHTML（可拓展超文本指标语言）所代替，因为它更接近xml，有更加严格的语法，比如标记符号必须要小些，必须要有结束标记，这一点适合xml相同 的。

    2.3 因为目前的浏览器还无法识别xml语言，但是xml确实是信息交换的标准和简易的方式，所以为了向xml过渡才出现XHTML.

  3. 那么xml与XHTML（或是html）有什么区别呢？

    3.1 XHTML（或是html）只能用于web（也就是网页中），而xml则不同，它是信息交换的标准语言，他可以跨平台进行信息的交流，比如电脑与手机

    3.2 xml的标记灵活多变，你可以起名字比如学生，但是XHTML和html是不行的，他两是写给浏览器看的，自己定义的就不会别识别，所以这样的符号是约定，比如百度，就是一个连接百度的超链接标记。
