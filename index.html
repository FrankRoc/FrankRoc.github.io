总结

1.编译工具

  • koala编译
    • Koala
    • 国人开发的LESS/SASS编译工具
    • 下载地址：http://koala-app.com/index-zh.html
  • Node.js库
  • 浏览器端使用

2.Less的注释

  • LESS中的注释
    • 可以书用css中的注释（/**/）
    • 也可以使用//注释
      • //编译时会自动过滤

3.变量

  • LESS中的变量
    • @开头
    • @test_width:300px

4.混合

  • 混合变量
    • 例如：.border{border:solid 10px red;}
  • 带参数的混合
    • .border-radius(@radius){css代码}
      • 可以设定默认值
      • border-radius(@radius:5px){css代码}

5.匹配模式

  • 匹配模式
    • 相当于js中的if判断语句
      • 但不全是
    • 满足条件后才能匹配
      • 可以有默认样式

6.运算

  • less中的运算
    • 任何数字、颜色或者变量都可以参与运算，运算应该被包裹在括号中。
      • 例如：+ - * /

7.嵌套规则

  • less中嵌套
    • 比较常用
    • 两种方法
      • & 对味蕾使用
        • hover 或 focus
      • 对连接的使用
        • &-item

8.@arguments变量

  • @arguments包含了所有传递进来的参数
  • 如果你不想单独处理每一个参数的话可以这么用

9.避免编译

  • 又是我们需要输出一些不正确的CSS语法或者使用一些LESS不认识的专有语法。
  • 要输出这样的值我们要在字符串前加一个一个 ~
    • 例如： width:~’calc(300px-30)’

10.!important

  • !important关键字
    • 会为所有混合所带来的样式添上！important

以上是我在慕课课网学习less即学即用课程整理的笔记。

git的一些基本指令

  • 查看当前文件内容列表（本地仓库各个文件名字以及格式）

  • 随时掌握工作区

  • 查询改文件改变情况

  • git log 显示从最近到最远的提交日志

git log –pretty=oneline 信息比较简单输出

  • 由过去穿越到未来的版本，查看所有日志

git reflog 记录你的每一次命令，也可以通过此方法找Commit id(非常不错的方法)。

  • 回退版本到

用HEAD表示当前版本，也就是最新的提交3628164…882e1e0（注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100。

  • 获取文件内容
  • 版本号（commit id）“首先要知道回退的版本号，其次版本号不一定要写全，但是也不能太少否则Git会匹配多个版本号造成混乱。

小结

  1. HEAD 指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令Git reset --hard commit_id.
  2. 在回退之前，可以用git log查看提交的历史
  3. 关闭窗口后回来查看版本，可以用git reflog来查看要回退到哪个版本

Git版本库里添加的时候，是分两步执行的：

  • 用git add把文件添加进去，实际上就是把文件修改添加到暂存区(index或者叫stage)；
  • git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支（Git自动为我们创建了唯一一个master分支）;
